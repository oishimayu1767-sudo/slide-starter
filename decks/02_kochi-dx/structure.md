## スライド詳細

---

### スライド 01 — `slide-title`

**ソース対応**: タイトル情報（テーマ・対象・形式）

```html
label  : 高知DXコミュニティ　ワークショップ
h1     : 工場データ取得と見える化（BI活用）
sub    : 共同作業型ワークショップ（もくもく会形式）
         対象：興味のある企業のみ参加
```

---

### スライド 02 — `slide-content` + `list`

**ソース対応**: 背景・課題

```html
h2: 背景・課題

list:
  - データは取得しているが活用できていない
  - 原価・稼働率・利益などが見えない
  - 現場データが感覚ベースで信頼性に課題
  - BI導入の目的が不明確なまま進んでいる
```

### スライド 05 — `slide-content` + `steps`

**ソース対応**: ワークショップ進め方

```html
section-label: 初回ワークショップ
h2: ワークショップ進め方

steps:
  01  チェックリスト形式で課題抽出
      （本文なし or 空白）
  02  答えられない項目を「困りごと」として認識
      （本文なし or 空白）
  03  参加者同士でディスカッション
      （本文なし or 空白）
  04  もくもく会形式で各社作業＋共有
      （本文なし or 空白）
```

> **実装メモ**: step-body の `p` は省略可。見出しだけでも成立する。

---

### スライド 06 — `slide-content` + `list`

**ソース対応**: チェックリスト例

```html
section-label: チェックリスト例
h2: チェックリスト例

list（各項目は「〜か」の形そのままで使う）:
  - 工程ごとの原価は把握できているか
  - 設備稼働率はリアルタイムで分かるか
  - 製品ごとの利益率は把握できているか
  - リードタイムは可視化できているか
```

> **実装メモ**: 「答えられない項目が困りごと」というコンテキストがスライド05にあるため、
> このリストはそのまま「問い」として提示できる。補足テキスト不要。

---

### スライド 07 — `slide-content` + `flow`（5段）

**ソース対応**: データ活用ステップ

```html
h2: データ活用ステップ

flow（5ステップ）:
  ① 目的整理  →  ② データ収集  →  ③ データ整備（クレンジング）
             →  ④ 可視化（BI活用）  →  ⑤ 改善アクション
```

> **実装メモ**: flow は横並び5段になるため文字を短くして対応。
> ステップ番号は `flow-step` の `h3` に ① 〜 ⑤ を入れ、`p` に短ラベルを入れる。
> もし窮屈であれば `steps`（縦リスト）に切り替えること。

### スライド 09 — `slide-content` + `cols-2` + `card`

**ソース対応**: 今後の展開

```html
h2: 今後の展開

cols-2:
  card 左:
    tag: 共有・連携
    h3: 各社事例の共有
    h3: 外部専門家の活用（乾氏など）
    h3: 外部コミュニティとの連携

  card 右:
    tag: 強化
    h3: DX推進支援の強化
```

> **実装メモ**: 4項目を2カラムに分割するのが最適。
> または `cols-2` を使わず `list` 4項目で1枚に収めても良い。

### スライド 10 — `slide-content` + `steps`

**ソース対応**: アクション

```html

  01  ワークショップ企画のブラッシュアップ
  02  専門家への協力打診
  03  次回日程の調整
  04  参加企業の募集
```

> **実装メモ**: アクション = 行動リストなので `steps` パターンが自然。
> 担当者・期日を追記できる余白を残すのが理想。

## 制作上の注意

1. **スライドカウンター**: `slide-counter` の分母を `11` に更新すること
2. **ロゴ**: `shared/logo/logo.png` があれば挿入、なければ省略
3. **チェックリストのスタイル**: `list` の箇条書き（丸ドット）をそのまま使用。
   チェックボックス風にする場合はインラインで `☐` を先頭に付ける方法があるが、
   ソースが「〜か」の問い形式なので丸ドットでも意図は伝わる。
4. **データ活用ステップの flow 5段**: 1280px幅で5ステップはやや窮屈。
   `flow-step` の `p` テキストを最小限にするか、`steps`（縦）に切り替えること。
